Visión general de componentes, flujos, módulos y API endpoints para el MVP de Lani.

Firebase Auth - Identidad y Seguridad

Flujo de autenticación, OTP y roles de usuario para el MVP de Lani.

  • OTP: Envío de códigos a móviles para login seguro.
  • Custom Roles: Admin, Cajero, Usuario, etc., manejados con Firebase Custom Claims.
  • Integración con Backend: Backend verifica roles y genera JWT para sesiones.
  • Flujo seguro: El usuario se autentica → Firebase valida → Backend controla acceso.

Módulos Clave del MVP

🛠️ Herramienta de Gestión (Gratuita)

  • POS Móvil: Flujo de venta rápido y fácil, con escáner de código de barras.
  • Gestión de Inventario: Registro de productos, control de stock y alertas de existencias bajas.
  • Registro de Gastos: Clasificación de egresos y control de deudas a proveedores.
  • Gestión de Cajas: Flujo de apertura, arqueo y cierre de caja.
  • Dashboard y Reportes Básicos: Resumen diario de ventas y gastos.

🛒 Marketplace Lani

  • Tienda Virtual Unificada: Publicación de productos con un solo clic.
  • Experiencia del Comprador: Interfaz web para explorar, buscar y comprar de forma segura.
  • Sistema de Reputación: Calificaciones y reseñas para construir confianza.
  • Verificación de Vendedores (KYB): Proceso de validación para garantizar la seguridad del marketplace.

Endpoints Críticos del BFF (.NET 8)

Autenticación y Seguridad

POST /api/v1/auth/otp/send Público

Inicia el flujo de login enviando un código OTP.

POST /api/v1/auth/otp/verify Público

Verifica el OTP y retorna un JWT para la sesión.

GET /api/v1/users/me Usuario

Obtiene los datos del perfil del usuario autenticado.

Gestión de Negocio (CRUD)

GET /api/v1/products Cajero/Admin

Lista todos los productos del negocio.

POST /api/v1/products Admin

Crea un nuevo producto en el inventario.

PUT /api/v1/products/{id} Admin

Actualiza la información de un producto existente.

GET /api/v1/expenses Cajero/Admin

Obtiene la lista de gastos registrados.

POST /api/v1/expenses Cajero/Admin

Registra un nuevo gasto para la caja/sucursal.

Operaciones de Venta y Sincronización

POST /api/v1/sync/actions Cajero/Admin

Endpoint principal para la sincronización offline.

POST /api/v1/cash-registers/{id}/open Cajero/Admin

Abre una caja con un monto inicial.

POST /api/v1/cash-registers/{id}/close Cajero/Admin

Cierra una caja y realiza el arqueo final.

Marketplace y Verificación

GET /api/v1/marketplace/products Público

Lista los productos disponibles en el marketplace para los compradores.

PUT /api/v1/marketplace/products/{id}/publish Admin

Publica o despublica un producto en el marketplace.

POST /api/v1/kyb/verifications Usuario

Inicia el proceso de verificación de negocio (KYB).
